# Playbook MVP PRD

## L1 Features
+ Playbook Builder
+ Clause Position Manager
+ Playbook Runner
+ Playbook Library
+ Custom Sub-Agents / Services

## L2 Features

### Playbook Builder
+ Create new playbook with document type selection (NDA, MSA, Employment, SaaS, etc.)
+ Name and describe playbook
+ Upload "gold standard" precedent document to bootstrap
+ AI-assisted clause extraction from uploaded examples
+ Manual clause selection from checklist
+ Add custom clauses
+ Set permissions (Personal / Team / Firm-wide / Client-specific)
+ Version history tracking

### Clause Position Manager
+ Define preferred position for each clause
+ Set acceptable range/alternatives
+ Define red lines (absolute rejections)
+ Specify fallback language
+ Add internal notes and business rationale
+ Set escalation paths
+ Link to policy documents

### Playbook Runner
+ Run playbook against uploaded document in app canvas
+ Real-time clause flagging with cited reasoning
+ Show deviation from preferred position
+ Generate suggested redlines
+ Side-by-side comparison (original vs suggested)
+ Export tracked changes

### Playbook Library
+ Browse all playbooks (filterable by type, owner, status)
+ Search playbooks
+ Clone existing playbook
+ Archive/delete playbook
+ View usage statistics
+ Share playbook with team

### Custom Sub-Agents / Services
+ *ClauseExtractorAgent*: Extract clauses from precedent documents
+ *PositionMatcherAgent*: Compare document clauses against playbook rules
+ *RedlineGeneratorAgent*: Generate precise redline suggestions
+ *RiskScorerAgent*: Score overall document risk based on deviations

---

## User Journey

### Step 1: Start with Context
"What type of agreement is this for?"
- Select document type
- Name the playbook
- Optional: Upload precedent document

### Step 2: Define Clauses to Track
"What provisions do you care about?"
- AI extracts clauses from uploaded examples, OR
- Manual selection from common clause checklist
- Add custom clauses as needed

### Step 3: Set Positions for Each Clause
"What's your stance on this clause?"
- Preferred Position
- Acceptable Range
- Red Lines
- Fallback Language
- Internal Notes

### Step 4: Add Guidance & Reasoning
"Why does this matter?"
- Business rationale
- Escalation path
- Policy document links

### Step 5: Set Permissions & Scope
"Who can use this playbook?"
- Personal / Team / Firm-wide / Client-specific

### Step 6: Test & Refine
"Try it on a real document"
- Upload sample contract
- Review flagged issues
- Adjust rules
- Mark as Active
